PAPER 11: SpecInfer: Accelerating Large Language Model Serving with
Tree-based Speculative Inference and Verification
1

Paper summary

The paper addresses the high latency and computational inefficiency of serving Large Language
Models (LLMs). Standard ”incremental decoding” is strictly sequential, which severely limits
parallelism, underutilizes GPUs, and creates major memory bottlenecks due to massive KeyValue (KV) caches. To overcome this, SpecInfer replaces traditional sequence-based speculative
execution with ”tree-based speculative inference and verification”. Rather than predicting a
single linear sequence, SpecInfer generates a multi-branch token tree representing multiple candidate paths. This structure drastically increases the probability of a correct match, enabling
the system to verify and accept multiple tokens per step.
These token trees are built using either an ”expansion-based” method from a single Small
Speculative Model (SSM) or a ”merge-based” method combining an ensemble of diverse SSMs.
For efficient verification, SpecInfer introduces ”tree-based parallel decoding,” which reuses a
shared KV cache via depth-first traversal and applies a ”topology-aware causal mask” to evaluate all tree branches simultaneously in a single GPU kernel launch. Generative quality is perfectly preserved using a ”multi-step speculative sampling” algorithm. Evaluations demonstrate
that SpecInfer matches baseline accuracy while accelerating distributed inference by 1.5–2.8x
over systems like vLLM, and reducing per-token latency in ”offloading-based LLM inference”
setups by 2.6–3.5x compared to FlexGen.
2

Contributions

This paper makes three primary contributions to the efficiency of large language model serving.
Firstly, it is the first paper to propose ”tree-based speculative inference” for LLMs, which
drastically increases the probability of successful speculation by replacing traditional singlesequence prediction with multi-branch token trees constructed from small speculative models.
Secondly, it presents a new technique for ”tree-based parallel decoding”, which dramatically
improves the performance and memory efficiency of the verification phase by leveraging a shared
KV cache, a depth-first traversal, and a ”topology-aware causal mask” to evaluate all candidate
branches simultaneously within a single GPU kernel.
Finally, the system offers perfectly preserved reliability for stochastic generation, saving a
large fraction of execution time while mathematically guaranteeing zero degradation in output
quality through the introduction of the ”multi-step speculative sampling” algorithm.
3

Problems

Despite its innovation, SpecInfer faces three scalability and operational limitations. Firstly,
because it relies on spare GPU resources, performance gains shrink significantly as batch sizes
increase. This non-linear scaling reduces its viability for high-throughput infrastructures.
Secondly, tree expansion relies on a static configuration rather than adapting dynamically
to the Small Speculative Model’s (SSM) confidence. This rigidity risks wasting resources or
missing target tokens, an issue explicitly left for future work.
Finally, caching keys, values, and attention scores for all unverified speculative branches
introduces unavoidable memory overhead. As LLMs demand massive context windows, this
overhead could exacerbate memory bottlenecks and restrict concurrent user capacity.
