PAPER 1: A High-Speed Load-Balancer Design with Guaranteed Per-ConnectionConsistency
1

Paper summary

This paper brings up the challenge of designing high-speed load balancers that can simultaneously achieve “uniform load distribution” and “per-connection-consistency (PCC)” in large-scale
data centers. PCC requires that “packets belonging to the same connection are forwarded to
the same server”. This becomes difficult in dynamic environments where servers are frequently
added or removed, as traditional stateless approaches (hashing) break existing connections,
while stateful approaches incur high memory overhead and are vulnerable to state-exhaustion
attacks.
To address this, the authors propose Cheetah, a load-balancing design that “completely decouples the load balancing logic from PCC support”. Instead of storing per-flow state, Cheetah
uses a stateless mechanism that encodes forwarding information directly into packets or reconstructs it deterministically. This enables the system to maintain PCC even under server churn
while remaining highly efficient.
Cheetah “can be deployed in both a stateless and stateful manner”. The stateless version
“encodes the selected server’s identifier into a cookie” into packets to preserve the connectionto-server mapping without storing state on the load balancer. The alternative stateful approach
uses a simplified memory design instead of complex cuckoo hashing. It maintains a stack of
available indices (ConnStack) and a connection table (ConnTable). For each new connection,
the load balancer assigns an index and stores the server mapping. This index is embedded in
the packet, enabling constant-time lookups for subsequent packets.
The system is evaluated using both software (DPDK) and hardware (Tofino switch) implementations. Results show that Cheetah guarantees 100% PCC while achieving performance
comparable to simple stateless load balancers. By enabling more advanced load-balancing policies such as least-loaded scheduling, it can “reduce the flow completion time by a factor of 2-3x”
compared to existing approaches. However, the evaluation is limited to specific environments
and may not fully capture deployment constraints in heterogeneous real-world networks.
2

Contributions

First, the paper introduces Cheetah, a load-balancing architecture that guarantees per-connection
consistency without requiring large per-flow state, enabling scalable and reliable operation.
Second, it separates server selection from connection consistency, allowing advanced loadbalancing algorithms while maintaining line-rate performance, significantly improving flow completion time.
Third, the authors implement Cheetah on both software and programmable hardware,
demonstrating that the design is practical and deployable in modern data center environments.
3

Problems

Despite its advantages, Cheetah has several limitations. First, the stateless design relies on
embedding information in packet headers, which assumes that intermediate network devices
preserve these fields. In practice, middleboxes such as NATs and firewalls may modify or strip
such information, potentially breaking correctness.
Second, the hardware implementation may face scalability constraints. Programmable
switches have limited memory and pipeline resources, and the complexity of Cheetah’s hashing
mechanisms may restrict the number of supported services and backend servers at scale.
Finally, while the stateful variant mitigates deployment limitations, it reintroduces dependence on memory. Under extreme workloads or distributed denial-of-service (DDoS) attacks,
this state can still be exhausted, “rapidly degrading performance” or leading to connection
drops.
