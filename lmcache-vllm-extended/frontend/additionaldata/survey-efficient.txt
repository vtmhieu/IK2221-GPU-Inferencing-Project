A Survey on Efficient Inference for Large Language Models
Authors: Zixuan Zhou, Xuefei Ning, Ke Hong, Tianyu Fu, Jiaming Xu, Shiyao Li, Yuming Lou, Luning Wang, Zhihang Yuan, Xiuhong Li, Shengen Yan, Guohao Dai, Xiao-Ping Zhang, Huazhong Yang, Yuhan Dong, Yu Wang
 
**Abstract**
LLMs achieve strong performance across tasks but impose substantial computational, memory, and energy costs that make deployment challenging in resource-constrained settings. This survey provides a comprehensive review of efficient LLM inference techniques, organised around three root causes of inefficiency — large model size, quadratic-complexity attention, and auto-regressive decoding — and a three-level taxonomy: data-level, model-level, and system-level optimisation. Unlike prior surveys, this work adds comparative experiments on representative methods in critical sub-fields including quantisation and serving systems, providing quantitative practical guidance alongside the literature review.
 
**1. Introduction and Preliminaries**
LLM inference proceeds in two stages. In the prefill stage, all input tokens are processed in one forward pass to generate the KV cache and the first output token. In the decode stage, tokens are generated one at a time, each requiring a full pass through the model with the growing KV cache. Three factors drive inference inefficiency: model size (LLaMA-2-70B requires 140 GB in FP16, demanding at minimum two A100 80 GB GPUs), attention complexity (quadratic in sequence length during prefill), and auto-regressive decoding (every decode step reloads all model weights from HBM, creating a memory-bandwidth bottleneck as the KV cache grows). Key metrics tracked in the survey are first-token latency, per-output-token latency, generation latency, model size, KV cache size, peak memory, token throughput, and request throughput.
 
**2. Taxonomy**
The survey organises existing work into three levels. Data-level optimisation modifies inputs or output structure without touching model weights, incurring little or no training cost and being lossless or near-lossless in quality. Model-level optimisation redesigns or compresses the model itself, typically requiring at least fine-tuning and accepting some quality trade-off. System-level optimisation improves the inference engine or serving infrastructure, is generally lossless, and requires no model training.
 
**3. Data-Level Optimisation**
Input compression techniques shorten prompts to reduce prefill cost. Prompt pruning methods remove tokens, sentences, or documents based on importance scores: Selective Context uses self-information as a pruning signal; LLMLingua applies a coarse-to-fine scheme using perplexity from a small proxy LLM combined with a budget controller; LongLLMLingua conditions perplexity on the input question to improve relevance. Prompt summary methods rewrite the prompt as a concise abstract rather than preserving tokens verbatim: RECOMP distils a lightweight abstractive compressor from large LLMs; SemanticCompression groups sentences by topic before summarising. Soft prompt-based compression encodes a fixed system or task prompt into a short sequence of learnable continuous tokens, avoiding repeated processing of the same prefix: PromptCompression and Gisting do this offline; AutoCompressors and ICAE extend the idea online. Retrieval-Augmented Generation (RAG) replaces large static contexts with dynamically retrieved relevant snippets, reducing both prompt length and the need for memorised knowledge.
 
Output organisation techniques exploit parallelism in generation. Skeleton-of-Thought (SoT) first generates a concise outline, then expands each point in parallel via batch inference, achieving up to 2.39x speedup on twelve LLMs while often improving answer quality. SGD extends this to a DAG structure for problems with partial ordering. APAR fine-tunes the model to emit fork tokens that trigger dynamic parallel decoding branches. SGLang provides a Python DSL for LLM programming that automatically identifies parallelisable generation calls and performs batch inference and KV cache sharing across them.
 
**4. Model-Level Optimisation**
Efficient structure design targets the two most expensive components: the FFN and the attention mechanism.
 
For the FFN, Mixture-of-Experts (MoE) architectures activate only a subset of FFN experts per token via a routing module, keeping quality high while reducing active compute. Key challenges include load imbalance (some experts handle far more tokens than others, wasting padded computation) and routing stability. Switch Transformers address load imbalance with an auxiliary balancing loss; Expert Choice inverts the assignment direction, letting each expert select its top-k tokens to guarantee perfect balance; StableMoE fixes the routing strategy after an initial learning phase to improve consistency between training and inference. Mixtral 8x7B demonstrates the practical payoff: it activates only 13B parameters during inference while matching or exceeding LLaMA-2-70B quality.
 
For attention, Multi-Query Attention (MQA) shares a single set of K and V heads across all query heads, substantially reducing KV cache size and memory bandwidth with minimal quality loss. Grouped-Query Attention (GQA) generalises this by grouping heads, providing a tunable quality-efficiency trade-off. Low-complexity attention variants target the O(n²) prefill cost: kernel-based methods (Linear Transformer, Performers) replace softmax with a kernel approximation that factors as ϕ(Q)(ϕ(K)ᵀV), reducing complexity to O(n); low-rank methods (Linformer, Luna) compress the token dimension of K and V to a fixed length k, reducing complexity to O(nk). Transformer alternates including State Space Models (Mamba, S4 family) and RWKV achieve linear complexity in sequence length via recurrent formulations, at the cost of reduced ability to attend globally.
 
Model compression techniques operate on pre-trained models, requiring only fine-tuning rather than pre-training from scratch.
 
Quantisation reduces the bit-width of weights and activations. Post-training quantisation (PTQ) is applied without retraining: GPTQ uses second-order gradient information to minimise quantisation error layer by layer; AWQ identifies salient weights that contribute disproportionately to output quality and preserves their precision; SmoothQuant migrates quantisation difficulty from activations (which have extreme outliers) to weights (which do not) via a mathematically equivalent per-channel scaling transformation, enabling efficient W8A8 quantisation. Quantisation-aware training (QAT) incorporates quantisation into the training loop to recover quality, at higher training cost. Comparative experiments in the survey show that 4-bit weight quantisation methods such as GPTQ and AWQ preserve most model quality while halving memory requirements relative to FP16, with AWQ showing better task performance in many settings.
 
Weight pruning reduces parameters by setting low-importance weights to zero. Structured pruning (e.g., SparseGPT, LLM-Pruner) removes entire heads, layers, or channels to enable hardware-friendly speedups, but risks larger quality drops. Unstructured pruning achieves higher compression ratios with lower quality impact but requires sparse computation support to realise actual speedups.
 
Knowledge distillation transfers capabilities from a large teacher to a smaller student. Black-box KD (e.g., Alpaca, Vicuna) uses teacher outputs as supervised training signal; white-box KD (e.g., MiniLLM) additionally aligns internal representations or output distributions for stronger transfer.
 
Dynamic inference techniques apply different computational budgets to different inputs. Early exit methods add classifier heads at intermediate layers and terminate generation when confidence is sufficient. Mixture-of-depths approaches skip certain layers for easy tokens. These reduce average compute without a fixed throughput penalty.
 
Structure optimisation techniques such as neural architecture search (NAS) and weight factorisation (e.g., SVD decomposition) reduce the parameter count or FLOPs of a model while attempting to preserve quality.
 
**5. System-Level Optimisation**
Inference engine optimisations improve how the model executes on hardware without changing model weights.
 
Graph and operator optimisation fuses multiple operations into single GPU kernels to reduce memory round-trips and kernel launch overhead. FlashAttention reorders the attention computation to tile across the memory hierarchy, avoiding materialising the full n×n attention matrix in HBM and achieving near-optimal memory bandwidth utilisation. CUDA Graphs capture kernel launch sequences to eliminate CPU-side scheduling overhead during decode.
 
Speculative decoding breaks the sequential token-generation bottleneck by using a small draft model to propose multiple tokens in parallel, then verifying them in a single forward pass of the large target model. Accepted tokens are kept; the first rejected token is replaced by the target model's prediction. Because verification is parallelisable, the latency approaches that of verifying one token regardless of draft length when the acceptance rate is high. Key design dimensions include the draft model source (a separate smaller model, a quantised version, a subset of layers, or n-gram lookup), the verification strategy, and adaptive draft length based on real-time acceptance rates. Comparative experiments show 1.5–3x decode speedup on suitable workloads with no quality loss.
 
KV cache offloading moves parts of the KV cache to CPU memory when GPU memory is exhausted, using prefetching and overlapping transfers with computation to hide latency. FlexGen extends this to disk offloading for extreme memory constraints, enabling inference on hardware that could not otherwise fit the model.
 
Serving system optimisations improve multi-request throughput and latency.
 
Memory management: vLLM introduces PagedAttention, which manages the KV cache using paged virtual memory analogous to OS memory paging. By allocating KV cache in fixed-size blocks rather than contiguous chunks, it eliminates fragmentation, enables efficient sharing of KV blocks across requests with common prefixes, and allows batch sizes limited only by total memory rather than worst-case padding.
 
Batching: Continuous batching (Orca) replaces static batching — where all requests in a batch must finish before new ones are admitted — with iteration-level scheduling. New requests are inserted into the batch as soon as a slot becomes free, dramatically improving GPU utilisation and throughput at the cost of minor per-request latency increases.
 
Scheduling: Systems such as Sarathi-Serve use chunked prefill to interleave prefill and decode work within a single batch, reducing time-to-first-token for waiting requests while maintaining high decode throughput. S3 and related work use output length predictors to approximate shortest-job-first scheduling and reduce head-of-line blocking.
 
Distributed systems: Tensor parallelism splits each layer across GPUs with all-reduce communication per layer; pipeline parallelism assigns layer groups to GPU stages. DistServe and Splitwise disaggregate prefill and decode stages onto separate GPU pools, allowing each to be independently scaled and scheduled. For long-context inference, sequence parallelism distributes the token dimension across GPUs.
 
**6. Application Scenarios and Conclusion**
The survey discusses how these techniques interact and combine across several deployment scenarios: on-device inference (where model compression and quantisation are primary), cloud serving (where throughput and cost-efficiency dominate), long-context tasks (where attention complexity and KV cache management are critical), and agent and multi-step pipelines (where output organisation and KV cache sharing across calls become relevant). The authors note that data-level, model-level, and system-level techniques are largely orthogonal and can be stacked: a quantised MoE model served with PagedAttention, continuous batching, and speculative decoding is a realistic and effective combination. Future directions highlighted include better hardware-software co-design, more accurate and lightweight output length predictors, richer support for diverse parallelism strategies in serving engines, and techniques tailored to emerging architectures such as SSMs and hybrid models.
