Optimizing LLM Inference: Fluid-Guided Online Scheduling with Memory Constraints
Authors: Ruicheng Ao, Gan Luo, David Simchi-Levi, Xinshang Wang
 
**Abstract**
LLM inference poses unique scheduling challenges because the KV cache grows dynamically during decoding, and memory overflow can trigger evictions that cascade into system-wide failures. Crucially, a system whose memory capacity exceeds the theoretical minimum can still become unstable under poor scheduling. This paper formalises LLM inference as a multi-stage online scheduling problem and develops a fluid dynamics approximation as a tractable upper-bound benchmark. Building on this, the authors introduce the WAIT (Waiting for Accumulated Inference Threshold) algorithm for the case where output lengths are known, and extend it to Nested WAIT for the realistic case where they are not. Both algorithms are proved asymptotically near-optimal in the heavy-traffic limit. Experiments on Llama-7B with an A100 GPU show superior throughput and reduced latency versus vLLM and Sarathi.
 
**1. Introduction**
LLM inference consists of a prefill phase, where all input tokens are processed in a single forward pass to initialise the KV cache, and a decode phase where output tokens are generated one at a time, each appending to the KV cache and growing its memory footprint linearly. This dynamic, growing resource demand means that classical scheduling techniques — including Shortest Job First, priority scheduling, and static batching — are inadequate: they assume fixed job sizes and cannot account for unpredictable memory growth. System-level engineering solutions such as vLLM and Sarathi improve throughput through batching and memory compression, but lack rigorous mathematical foundations. This paper bridges operations research and machine learning by establishing a formal scheduling framework with theoretical guarantees under memory constraints.
 
**2. Model**
The model considers a single GPU processing prompts that arrive as independent Poisson processes, each with a known prompt type characterised by an input length and an output length. Prompt processing proceeds in two phases: stage 0 (prefill), consuming KV cache memory equal to the input length, and stages 1 through l' (decode), each generating one output token and incrementing memory by one unit. A batch may mix prompts from different types and stages; iteration time scales linearly with total KV cache memory in the batch, a relationship validated empirically on Llama-7B and Llama-13B. The memory constraint requires total KV cache usage to remain within GPU capacity C at every iteration. The optimisation objective is to maximise expected throughput subject to constraints on average latency and time-to-first-token (TTFT).
 
**3. Fluid Dynamics and Equilibrium**
The paper develops a fluid dynamics model that approximates the stochastic, discrete arrival and processing dynamics with a continuous, deterministic flow in equilibrium. In this idealised steady state, the arrival rate for each prompt type equals its completion rate and memory usage stabilises. For a single prompt type, the equilibrium number of active prompts, memory usage, and throughput are derived in closed form. The key condition for stability is that the product of arrival rate and per-prompt processing time must be strictly less than one; if not, expected latency grows linearly with time and the system is provably unstable. The fluid equilibrium throughput — equal to the sum over all types of arrival rate times (1 + output length) — is shown to be an upper bound for any online scheduling policy: no algorithm can exceed this benchmark in expectation. This benchmark guides both the theoretical analysis and the design of practical algorithms.
 
**4. The WAIT Algorithm (Known Output Lengths)**
WAIT accumulates prompts of each type in per-type queues and delays processing until the inventory of each type reaches a threshold derived from the fluid equilibrium. When thresholds are met, it forms a batch by selecting a fixed number of prompts at each processing stage for each qualifying type, processes the batch, and repeats. KV caches of waiting prompts are preserved on-GPU to avoid recomputation overhead. The threshold conditions are chosen so that the expected number of new arrivals during one iteration does not exceed the threshold, keeping the system close to fluid equilibrium. Theorem 1 proves that under appropriate threshold conditions, the throughput gap between WAIT and the fluid optimum vanishes at rate O(1/sqrt(T)) in the boundary case, and at O(1/T) with bounded O(1) latency and TTFT when thresholds are strictly satisfied. A matching lower bound (Proposition 3) shows that O(sqrt(T)) latency growth is unavoidable for any non-predictive policy when memory is tight. First-Come-First-Served (FCFS) scheduling is shown to incur a constant throughput gap Omega(1) relative to optimal, because it frequently preempts ongoing prompts to admit new arrivals, causing recomputation and memory thrash.
 
**5. Nested WAIT (Unknown Output Lengths)**
When output lengths are unknown at arrival, assuming worst-case lengths is highly inefficient: with two types having output lengths 1 and 10,000, treating all as length 10,000 wastes up to 5,000x memory. Instead, Nested WAIT organises processing into m nested segments corresponding to cumulative output length milestones. All prompts enter segment 1; those that complete at the first milestone exit, while longer prompts naturally advance to segment 2, and so on. Prompts are thus classified by their actual output length as inference proceeds rather than predicted upfront. Thresholds govern when each segment is activated, and KV caches of waiting prompts in later segments are held on-GPU. Proposition 5 establishes that when memory equals the equilibrium minimum, any non-predictive policy suffers an unavoidable constant throughput gap — demonstrating that additional memory is necessary for near-optimality without output length information. To handle this, Theorem 2 establishes that Nested WAIT achieves asymptotic near-optimality under a slightly larger memory budget that includes a logarithmic safety buffer of O(log(m * zeta * S / delta)), which provides high-probability protection against memory overflow from initially unknown output lengths. The dependence of the required memory buffer on the number of prompt types is only logarithmic, so the algorithm scales efficiently even with many distinct output length categories.
 
**6. Numerical Experiments**
Experiments use Llama2-7B on a single A100 GPU with both synthetic workloads and real-world datasets. WAIT and Nested WAIT are compared against vLLM and Sarathi (a chunked-prefill inference engine). Across settings, the WAIT family achieves higher throughput and lower average latency than both baselines. On synthetic data, WAIT's performance closely tracks the fluid benchmark, validating the theoretical analysis. Real-world results confirm practical gains in latency and TTFT, supporting the claims of near-optimality in heavy-traffic conditions.
 
**7. Related Work and Conclusion**
Classical online scheduling and queueing theory provide foundational tools — including fluid models and heavy-traffic analysis — but cannot directly handle LLM inference's combination of dynamic KV cache growth, multi-phase processing, and stochastic arrivals. System-level LLM inference work (Orca, vLLM, Sarathi, DistServe, Splitwise) focuses on engineering efficiency without theoretical guarantees. Concurrent theoretical work by Li et al. (2025) studies work-conserving scheduling for throughput optimality but does not address KV cache memory constraints or latency metrics such as TTFT. This paper's contribution is a theoretically grounded framework — connecting fluid queuing models to practical threshold-based algorithms — that achieves provable near-optimality on throughput, latency, and TTFT simultaneously under realistic memory constraints.
